Graph Coloring

Einführung

In der Graphentheorie ist die Graphfärbung ein Spezialfall der Graphbeschriftung; Hierbei handelt es sich um eine Zuordnung von Beschriftungen, die traditionell "Farben" genannt werden, zu Elementen eines Diagramms, die bestimmten Einschränkungen unterliegen. In seiner einfachsten Form können die Scheitelpunkte eines Diagramms so gefärbt werden, dass keine zwei benachbarten Scheitelpunkte dieselbe Farbe haben. Dies wird als Scheitelpunktfärbung bezeichnet. In ähnlicher Weise weist eine Kantenfärbung jeder Kante eine Farbe zu, so dass keine zwei benachbarten Kanten dieselbe Farbe haben, und eine Flächenfarbe eines ebenen Graphen weist jeder Fläche oder Region eine Farbe zu, so dass keine zwei Flächen, die eine Grenze haben, dieselbe Farbe haben Farbe.

In ähnlicher Weise weist eine Kantenfärbung jeder Kante eine Farbe zu, so dass keine zwei benachbarten Kanten dieselbe Farbe haben, und eine Flächenfarbe eines ebenen Graphen weist jeder Fläche oder Region eine Farbe zu, so dass keine zwei Flächen, die eine Grenze haben, dieselbe Farbe haben Farbe.

In diesem Projekt behandeln wir das Problem der Scheitelpunktfärbung mit verschiedenen Algorithmen in einer Reihe von zufällig generierten Graphen. Wir lösen das Minimierungsproblem (ermitteln Sie die minimale Anzahl von Farben, die zum Färben eines Diagramms verwendet werden können) und das Maximierungsproblem (finden Sie den maximalen Prozentsatz des Diagramms, der mit x Farben eingefärbt werden kann). Wir erstellen auch eine Sudoku-Solver-Anwendung, die zur Lösung Graph-Farben verwendet. Wir vergleichen diese verschiedenen Algorithmen auch mit einem riesigen Datenbestand aus der realen Welt, um die Zeit- und Farbeffizienz zu vergleichen.

Anwendungen

Scheduling

Vertex-Modelle für eine Reihe von Planungsproblemen. [30] In der saubersten Form muss ein bestimmter Satz von Jobs Zeitfenstern zugewiesen werden, jeder Job erfordert einen solchen Slot. Jobs können in beliebiger Reihenfolge eingeplant werden. Jobpaare können jedoch in einem Konflikt miteinander stehen, dass sie möglicherweise nicht demselben Zeitfenster zugewiesen werden, z. B. weil beide auf einer gemeinsam genutzten Ressource basieren. Der entsprechende Graph enthält einen Scheitelpunkt für jeden Job und eine Kante für jedes in Konflikt stehende Jobpaar. Die Farbzahl des Diagramms ist genau die minimale Zeitspanne, die optimale Zeit, um alle Jobs ohne Konflikte zu beenden.

Details zum Planungsproblem definieren die Struktur des Diagramms. Beim Zuweisen von Flugzeugen zu Flügen ist das resultierende Konfliktdiagramm beispielsweise ein Intervalldiagramm, sodass das Farbproblem effizient gelöst werden kann. Bei der Bandbreitenzuweisung für Radiosender ist der resultierende Konfliktgraph ein Einheitsdiskgraph, sodass das Farbproblem 3-approximierbar ist.

Registerzuteilung

Ein Compiler ist ein Computerprogramm, das eine Computersprache in eine andere übersetzt. Um die Ausführungszeit des resultierenden Codes zu verbessern, ist eine der Techniken der Compiler-Optimierung die Registerzuweisung, bei der die am häufigsten verwendeten Werte des kompilierten Programms in den schnellen Prozessorregistern gespeichert werden. Idealerweise werden Werte Registern zugewiesen, sodass sie bei ihrer Verwendung alle in den Registern gespeichert werden können.

Der Lehrbuchansatz für dieses Problem besteht darin, es als ein Diagrammfärbungsproblem zu modellieren. [31] Der Compiler erstellt ein Interferenzdiagramm, wobei Vertices Variablen sind und eine Kante zwei Vertices verbindet, wenn sie gleichzeitig benötigt werden. Wenn der Graph mit k Farben gefärbt werden kann, kann jeder Satz von Variablen, der gleichzeitig benötigt wird, in höchstens k Registern gespeichert werden.

Sudoku-Puzzle

Ein Sudoku-Puzzle kann als Farbproblem der Grafik ausgedrückt werden. Ziel ist es, eine 9-Färbung eines bestimmten Graphen bei einer teilweisen 9-Färbung zu konstruieren.
